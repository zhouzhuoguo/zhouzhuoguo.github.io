Current Research

💻 Chip performance evaluation modeling

Simulation of mainstream AI models(Diffusion, Transformer, CNNs) on customized chip

  • Accurate:
    • Behaves like hardware on each clock rising edge
    • Instantaneous EMA, utilization, power, etc.
  • Complete:
    • Covers all modules: interfaces, NoC, memory management (allocate, release, conflict)
    • Models real dataflow: on-chip, off-chip, and inter-module communication
  • Scalable: Speed vs Granularity
    • Cycle-accurate (1 cycle/step): ~ 6 hours
    • Stride mode (100 cycles/step): ~10 minutes (about 15% deviation)

💼 Latest progress about modeling

  • Decoupled:
    • Decoupled the model into the main model, editable HW_Setting and ARCH. It is convenient to simulate different network models.
  • Runtime:
    • Take the diffusion model as an example, from 50h+ to <10min
  • Dataflow redesigned:
    • Now this model can support cross-layer routing & multi-source inputs.
  • Process-aware HW model:
    • The chip manufacturing process can be configured. Besides, runtime, energy, power, off-chip energy and access count can be reported
  • Bit-width parameter:
    • The data bit width is editable
  • Local_attention module:
    • It can aggregate per-stage metrics and scale for large repeated stages.

⛳️ Future works about modeling

    • Further improve the function of the model, such as adding (non) linear module (UFP) and transpose.
    • Add and evaluate technique options
    • Evaluate more SOTA algorithm model such as Flux, Video Diffusion Models
